ExynosTools V3.0 STABLE (2026-04-09)

Core goals:
- Stable Vulkan Layer architecture (no custom ICD bridge mixing).
- Safe dispatch chaining for Android Vulkan loader.
- Xclipse-focused behavior without forcing unsupported features.

What was hardened:
- Proper vkGetDeviceProcAddr routing and separate device dispatch path.
- Command pool and command buffer lifecycle tracking:
  - vkCreateCommandPool
  - vkDestroyCommandPool
  - vkResetCommandPool
  - vkBeginCommandBuffer
  - vkResetCommandBuffer
  - vkAllocateCommandBuffers
  - vkFreeCommandBuffers
- Cleanup of stale mappings on pool/device destroy.
- Cleanup of per-command-buffer staging buffers and descriptor sets on:
  - vkResetCommandPool
  - vkBeginCommandBuffer (implicit reset path)
  - vkResetCommandBuffer
  - vkFreeCommandBuffers
- One-time warnings for missing command buffer mapping in copy hooks.

Xclipse-specific behavior:
- Runtime detection of Samsung/Xclipse GPUs.
- Runtime capability readback:
  - geometryShader
  - tessellationShader
  - transformFeedback (when available via Features2 chain)
- BCn interception path only kept active on Xclipse devices.
- BCn virtualization restricted to supported 2D image paths.
- VMA now uses the real Vulkan API version exposed by the physical device.
- BC4/BC5 replacement formats can use R8/RG8 variants when supported to reduce VRAM waste.
- Virtualized image creation now clears VK_IMAGE_CREATE_MUTABLE_FORMAT_BIT.
- Virtualized image format lists are sanitized when VkImageFormatListCreateInfo is present.
- vkCmdCopyImage, vkCmdCopyImage2 and vkCmdCopyImage2KHR now have a real routed path for virtual images.

Build and packaging:
- Android arm64 build validated from scripts/configure_android_local_repos.ps1.
- Release packaging script added to emit standard driver structure.
